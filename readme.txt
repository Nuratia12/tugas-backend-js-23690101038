1. Perbedaan Front-End dan Back-End

Analogi: Toko

Front-End: Etalase & kasir → yang dilihat dan digunakan pengguna

Atur tampilan (warna, tombol, form)

Interaksi pengguna (klik, isi form)

Kenyamanan penggunaan

Back-End: Gudang & pembukuan → yang bekerja di belakang layar

Mengolah data pengguna

Menyimpan & mengambil data dari database

Menjaga keamanan dan logika sistem

Kesimpulan:

Front-End = tampilan & interaksi

Back-End = proses & keamanan

2. Perbedaan GET dan POST

Analogi: Toko online

GET: Data terlihat di URL, cocok untuk mengambil data, kurang aman

POST: Data tersembunyi, cocok untuk mengirim data penting (login, password, simpan database)

Kapan pakai POST: Saat mengirim data pribadi atau menyimpan data ke database

3. Mengapa validasi input tetap di Back-End

Alasan:

Front-End bisa dimanipulasi → seperti pelanggan masuk lewat pintu belakang

Back-End mencegah data palsu & serangan → seperti brankas yang aman meski satpam lengah

Analogi singkat:

Front-End = satpam

Back-End = brankas
